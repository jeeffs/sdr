Configuração do Firebase necessária

Para que o app funcione em múltiplos dispositivos, você precisa criar um banco de dados gratuito no Firebase. Siga os passos:

1. Acesse console.firebase.google.com e clique em Adicionar projeto.
2. Escolha um nome (ex: servico-rua), desative o Google Analytics e clique em Criar projeto.
3. No menu lateral, clique em Realtime Database → Criar banco de dados → escolha o local us-central1 → Iniciar no modo de teste.
4. Ainda no menu lateral, clique em ⚙️ Configurações do projeto → aba Geral → role até Seus apps → clique em </> (Web) → registre o app → copie o objeto firebaseConfig.
5. Cole o conteúdo do objeto firebaseConfig abaixo e clique em Salvar.

Importar Planilhas Históricas

Selecione a pasta inteira do ano (ou arquivos individuais) para importar.
Padrão do arquivo: NOME PROFILE TIPO.xlsx — ex: CAIO FILIAL MANUTENÇÃO.xlsm
Planilhas Prestador são a base de dados dos prestadores regulares. Planilhas Admin são a base de dados do master (Jefferson).
Planilhas PRESTADOR são lidas apenas como referência para cálculo do bônus (não geram registros).
Estrutura recomendada: 2025 / 01 / V1 / CAIO FILIAL MANUTENÇÃO.xlsm (técnicos) e 2025 / 01 / V3 / JEFFERSON FILIAL MANUTENÇÃO.xlsm (master) e 2025 / 01 / V2 / ... (referência bônus)

"Pasta inteira" lê todos os .xlsx/.xlsm dentro da pasta e subpastas automaticamente.
